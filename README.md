# Création d'un jeu Supémon en C
Le but de ce jeu est de créer un jeu qui se calque sur le Jeu Pokémon en langage C.
Le joueur entre son prénom si c'est la première fois qu'il joue ou bien il peut charger un fichier.
Ensuite il doit choisir entre 3 supémons (qui sera son supémon de départ). Il a ensuite un choix à faire : Il doit choisir entre aller dans le foret (un combat avec un supémon l’attend), dans le magasin où il
peut acheter des items qui peuvent l’aider, dans le ‘supémon centre’ ou il peut guérir
ses supemons ou il peut décider de quitter le jeu.
S'il va dans le foret un combat tour a tour (de style pkémon) est déclencher avec un supémon aléatoire. Le
Supemon qui a la plus grande vitesse commence le tour : celui-ci peut jouer un de ses
‘move’, des moves peuvent infliger des dégâts ou booster ses statistiques. Pendant le
tour du joueur il peut choisir entre : utiliser les ‘moves’ de son supémon, changer de
supémon, essayer de capturer le supémon adverse (pas garantit), utiliser un objet qu’il
a acheté dans le magasin où il peut essayer de fuir (pas garantit).
S'il gagne le match, il reçoit de l’expérience. Pour monter de niveaux le joeur doit dépasser la limite d'xp. s'il monte de niveau tous ces statistiques augmentent de 30% et il recoit de l'argent, argent qu'il peut utiliser pour acheter des items dans le magasin.
Une fois chaqun de ces 'actions' finis,le joueur retourne au menu qui lui demande où il veut aller.
Les itmes disponible dans le magasin sont : une potion (qui guérit de 3 hp sur le supémon choisis), 
une super-potion (qui guérit de 5 hp sur le supémon choisis), un 'rare candy' qui augmente le niveau du joeueur. Chaqun de ces objets ont un prix et le joueur ne peut les acheter uniquement si il n'a le montant suffisant.

### Ce que ce jeu m'a appris
Il s'agit d'un long projet que j'ai fait seul. Ce projet m'a beaucoup appris sur les pointeurs et les
structures (le joueur, les différents supémons...). J'ai dû apprendre à bien m'organiser car il y a beaucoup de code et a ne pas me perdre dedans. J'ai du bien séparer mon code en fontions adéquate pour 
que le joueur navigue comme établi aux différentes partie du jeu. Ce projet a été un gros challenge pour moi tant bien par sa longueur, sa compléxité, et le temps donner pour ce projet.
